# 공지사항 게시판 구현 기획안

## 1. 개요
인트라넷 대시보드(`pages/intranet.php`) 내에 사내 공지사항을 공유할 수 있는 게시판 섹션을 추가합니다. 관리자는 공지를 작성하고, 사용자는 최신 공지를 확인할 수 있습니다.

## 2. 데이터베이스 설계 (Database)

### 테이블: `notices` (신규 생성)
| 필드명 | 타입 | 설명 |
|---|---|---|
| `id` | INT (PK, AI) | 고유 ID |
| `title` | VARCHAR(255) | 공지 제목 |
| `content` | TEXT | 공지 내용 (HTML 지원 여부 결정 필요) |
| `author_id` | INT | 작성자 ID (`users` 테이블 참조) |
| `is_important` | TINYINT(1) | 중요 공지 여부 (상단 고정) |
| `view_count` | INT | 조회수 |
| `created_at` | TIMESTAMP | 작성일시 |
| `updated_at` | TIMESTAMP | 수정일시 |

## 3. 기능 명세 (Feature Specs)

### 3.1 공지사항 리스트 (인트라넷 메인)
- **위치**: 인트라넷 상단 또는 중앙 (`<!-- 공지사항 게시판 섹션 -->` 위치)
- **표시 항목**: 제목, 작성일, 작성자(선택), 중요 표시(아이콘/색상)
- **기능**:
    - 최신순 정렬 (중요 공지 최상단)
    - 클릭 시 상세 내용을 모달(Modal) 팝업으로 표시
    - '더보기' 버튼으로 전체 목록 페이지(또는 스크롤) 이동

### 3.2 공지사항 작성/수정 (관리자 전용)
- **진입**: 리스트 상단 '글쓰기' 버튼 (권한 체크)
- **입력 폼**: 제목, 내용(Textarea), 중요 공지 체크박스
- **저장**: `process/notice_process.php`를 통해 DB 저장

### 3.3 공지사항 상세 보기
- 모달 창 띄우기
- 수정/삭제 버튼 제공 (작성자 본인 또는 슈퍼관리자)

## 4. UI/UX 디자인
- **스타일**: 기존 Payday 테마(`style.css`)를 유지하며 깔끔한 카드(Card) 또는 테이블 형태
- **반응형**: 모바일에서도 리스트 및 상세 내용 확인 가능하도록 구현

## 5. 구현 파일 목록
1.  **DB Migration**: `notices` 테이블 생성 SQL
2.  **Frontend**:
    - `pages/intranet.php`: 리스트 UI 및 모달 추가
    - `css/style.css`: 게시판 관련 스타일 추가
    - `js/notice.js` (또는 인라인): 모달 제어 및 AJAX 처리
3.  **Backend**:
    - `process/notice_process.php`: 등록/수정/삭제/조회(AJAX) 처리

## 6. 승인 요청 사항
위 기획 내용으로 진행하시겠습니까? 특히 **'중요 공지' 상단 고정** 기능과 **모달 팝업** 형태의 상세 보기가 적절한지 확인 부탁드립니다.
